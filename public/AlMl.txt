## 2. The 7-Step Machine Learning Process

Building a predictive model follows a rigorous lifecycle to ensure accuracy and relevance:

1. **Define Objective:** Clearly state what you are trying to predict (e.g., "Will it rain tomorrow?").
2. **Data Gathering:** Collecting information via manual entry, web scraping, or public datasets (e.g., humidity, temperature).
3. **Data Preparation:** Cleaning "dirty" data by handling missing values, duplicates, and inconsistencies. This is often the **most time-consuming step**.
4. **Exploratory Data Analysis (EDA):** Brainstorming and finding correlations (e.g., lower temperature correlating with higher rain probability).
5. **Building the Model:** Splitting data into **Training** (to build the model) and **Testing** (to validate it) sets.
6. **Model Evaluation:** Using the test set to calculate accuracy and fine-tuning parameters.
7. **Predictions:** Deploying the model to solve the initial objective.

## 3. The Core of Deep Learning: The Perceptron

Deep Learning mimics the human brain using **Artificial Neural Networks**. The basic building block is the **Perceptron** (an artificial neuron).

### How a Perceptron Works:

| Component | Function |
| --- | --- |
| **Inputs ($x_1, x_2, ... x_n$)** | The raw data points fed into the system. |
| **Weights ($w_1, w_2, ... w_n$)** | Values that represent the importance or "strength" of each input. |
| **Summation Function** | Multiplies each input by its weight and adds them together: $\sum (x_i \cdot w_i) + \text{bias}$. |
| **Activation Function** | A "gatekeeper" (e.g., Sigmoid, ReLU) that maps the sum to an output (0 or 1). |

## 1. Deep Learning: Beyond the Single Perceptron

The speaker explains that while a single-layer perceptron is great for linear problems, real-world data is rarely that simple.

* **Multi-Layer Perceptron (MLP):** Unlike a single-layer model, an MLP has **hidden layers**. These layers allow the model to handle high-dimensional and non-linearly separable data.
* **Backpropagation:** This is the "learning" mechanism. The model starts with random **weights**, calculates an output, and compares it to the actual result. If the error is high, it "propropagates" back through the layers to update the weights and minimize the error.
* **Bias:** A value that allows the activation function curve to shift, helping the model fit the data more precisely.

# Part 1: Natural Language Processing (NLP)

## 1. Tokenization

Tokenization means breaking a sentence into smaller parts called **tokens** (usually words).

Example:

Sentence:

> "Machine learning is powerful"

Tokens:

* Machine
* learning
* is
* powerful

Why?

* Easier to analyze single words than full sentences.
* Fundamental preprocessing step in NLP.

---

# 2. Stemming

Stemming reduces words to their root/base form by removing prefixes or suffixes.

Example:

* detection
* detected
* detecting
* detections

All become:

> detect

Purpose:

* Simplifies text analysis
* Treats similar words as the same word

Problem:
Stemming can sometimes remove meaningful parts incorrectly.

Example:

* recap → cap

But “recap” and “cap” do not mean the same thing.

---

# 3. Lemmatization

Lemmatization is a smarter version of stemming.

Instead of randomly cutting words, it:

* understands word meaning
* performs morphological analysis
* returns a valid dictionary word

Example:

* gone
* going
* went

All become:

> go

Difference from stemming:

* Stemming may produce invalid words
* Lemmatization produces meaningful words

---

# 4. Stop Words

Stop words are very common words that usually do not add significant meaning.

Examples:

* the
* is
* how
* when
* why
* yes
* no

Example Search:
Instead of:

> "how to make strawberry milkshake"

Use:

> "strawberry milkshake recipe"

Benefits:

* Reduces computation
* Focuses on important keywords
* Improves NLP performance

---

# 5. Document Term Matrix (DTM)

A Document Term Matrix stores word frequencies in documents.

Example:

| Word | Doc1 | Doc2 | Doc3 |
| ---- | ---- | ---- | ---- |
| this | 1    | 1    | 1    |
| is   | 1    | 1    | 0    |
| fun  | 1    | 0    | 0    |

Purpose:

* Understand important words
* Analyze word frequency
* Used in text classification and NLP

# Regression in Machine Learning

Regression is a machine learning technique used to **predict continuous numerical values**.

Examples:

* Predicting house prices
* Predicting stock prices
* Predicting salary
* Predicting rainfall
* Predicting temperature

The goal is to understand the relationship between:

* **Independent variables (X)** → input features
* **Dependent variable (Y)** → output value to predict

Example:

* Inputs: house size, number of rooms, location
* Output: house price
